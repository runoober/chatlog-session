# Changelog

## [0.19.0] - 2026-03-10

### Added

#### 消息顺序稳定性工具

- 新增消息时间归一化与有序合并工具，解决历史加载、增量刷新、缓存混合场景下的时序稳定性问题。
- 新增批次顺序断言与校验辅助逻辑，便于在调试模式下快速发现逆序或错序数据。

### Changed

#### 聊天消息加载流程重构

- `chat` store 的消息加载链路改为“标准化批次 + 有序合并”模式：
  - 统一对 API 返回批次做时间归一化。
  - 根据批次与当前列表的时间关系自动选择 prepend / append / merge。
  - 在关键路径增加有序断言，确保消息列表始终保持时间顺序一致。
- `chatlog` API 调整以配合新的时间窗口与顺序处理策略。
- 升级应用版本号到 `0.19.0`。

### Technical Details

- **新增文件**:
  - `src/stores/chat/utils.ts` - 消息归一化、顺序断言、有序合并等工具函数
- **修改文件**:
  - `src/stores/chat.ts` - 重构消息加载/合并主流程
  - `src/api/chatlog.ts` - 适配新的消息加载参数与顺序处理
  - `package.json` - 版本号更新至 `0.19.0`

### Notes

- 本版本重点是“数据一致性与顺序稳定性”，对用户可见变化较少，但显著提升了历史消息加载的可靠性。
